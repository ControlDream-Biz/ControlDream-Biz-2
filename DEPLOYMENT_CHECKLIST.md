# 部署检查清单 (Deployment Checklist)

本文档列出了每次上线前必须完成的所有检查项，确保系统的安全性、稳定性和性能。

## 📋 部署前检查 (Pre-Deployment)

### 1. 代码质量检查

- [ ] 所有代码已提交到 Git 仓库
- [ ] 通过 `pnpm lint` 检查，无 ESLint 错误
- [ ] 通过 `pnpm ts-check` 检查，无 TypeScript 错误
- [ ] 通过 `pnpm build` 构建，无构建错误
- [ ] 所有测试用例通过
- [ ] 代码审查已完成并获得批准

### 2. 安全检查

#### 2.1 源码保护
- [ ] 确认 `next.config.ts` 中已配置代码混淆
- [ ] 确认 `productionBrowserSourceMaps: false` 已启用
- [ ] 确认 `config.devtool` 未设置为 `false`（避免 Turbopack 错误）

#### 2.2 授权域名
- [ ] 确认 `NEXT_PUBLIC_AUTHORIZED_DOMAIN` 环境变量已设置
- [ ] 确认授权域名列表正确（支持多个域名，用逗号分隔）
- [ ] 确认域名格式正确（不带协议，如 `example.com,www.example.com`）

#### 2.3 环境变量
- [ ] `DATABASE_URL` 已设置且格式正确
- [ ] `NEXT_PUBLIC_APP_VERSION` 已设置
- [ ] `NEXT_PUBLIC_SECURITY_ENABLED` 已设置为 `true`
- [ ] 敏感信息（如 API 密钥）已配置
- [ ] 开发环境变量（如 `DEV` 标志）已在生产环境移除

### 3. 依赖检查

- [ ] 运行 `pnpm install` 安装最新依赖
- [ ] 检查 `package.json` 中是否有未使用的依赖
- [ ] 检查是否有安全漏洞（运行 `pnpm audit`）
- [ ] 所有依赖版本已锁定（使用 `package-lock.json`）

## 🔨 构建检查 (Build Verification)

### 4. 构建验证

运行构建脚本：`bash scripts/build.sh`

#### 4.1 构建输出检查
- [ ] 构建成功，无错误信息
- [ ] `.next` 目录已生成
- [ ] `BUILD_ID` 文件存在
- [ ] `static` 资源目录存在

#### 4.2 源码保护验证 ⚠️ 关键
- [ ] **无 .map 文件**：运行 `find .next -name "*.map"` 应无结果
- [ ] **代码已混淆**：检查 `.next/server/app` 目录，不应有 `console.log`
- [ ] **文件大小合理**：构建产物大小符合预期

#### 4.3 代码混淆验证
- [ ] 函数名已混淆（非原始名称）
- [ ] 类名已混淆（非原始名称）
- [ ] 属性名已混淆（以下划线开头）
- [ ] 注释已删除
- [ ] 代码已压缩（无空格和换行）

## 🚀 部署检查 (Deployment)

### 5. 服务器环境

- [ ] 服务器操作系统版本正确
- [ ] Node.js 版本符合要求（>= 20）
- [ ] 磁盘空间充足（至少 5GB 可用）
- [ ] 内存充足（至少 2GB 可用）
- [ ] 端口 5000 未被占用

### 6. 数据库

- [ ] 数据库服务运行正常
- [ ] 数据库连接字符串正确
- [ ] 数据库表结构已创建
- [ ] 数据库索引已创建
- [ ] 数据库备份已完成

### 7. 文件部署

- [ ] 所有构建文件已上传到服务器
- [ ] `.next` 目录权限正确（755）
- [ ] `public` 目录权限正确（755）
- [ ] 环境变量文件已上传（`.env.local` 或 `.env.production`）
- [ ] 环境变量文件权限正确（600）

### 8. 服务启动

- [ ] 运行 `pnpm start` 启动服务
- [ ] 服务启动成功，无错误日志
- [ ] 端口 5000 正常监听
- [ ] 进程运行稳定（无崩溃）

## ✅ 上线后检查 (Post-Deployment)

### 9. 功能验证

#### 9.1 基础功能
- [ ] 首页可以正常访问（HTTP 200）
- [ ] 所有页面可以正常访问
- [ ] 页面加载速度正常（< 3秒）
- [ ] 图片资源正常加载
- [ ] 静态资源正常加载（CSS、JS）

#### 9.2 API 接口
- [ ] `/api/chat` 接口正常响应
- [ ] `/api/analytics` 接口正常响应
- [ ] `/api/admin/*` 接口正常响应
- [ ] 接口响应时间正常（< 1秒）
- [ ] 接口返回数据正确

#### 9.3 多语言支持
- [ ] 语言切换功能正常
- [ ] 所有语言内容正确显示
- [ ] 翻译文件完整

#### 9.4 安全功能
- [ ] 安全响应头已启用
- [ ] CSP 策略正确配置
- [ ] HSTS 已启用
- [ ] X-Frame-Options 已设置
- [ ] 源码保护正常（无 .map 文件可访问）

### 10. 性能检查

- [ ] LCP (Largest Contentful Paint) < 2.5s
- [ ] FID (First Input Delay) < 100ms
- [ ] CLS (Cumulative Layout Shift) < 0.1
- [ ] TTI (Time to Interactive) < 3.5s
- [ ] 网页性能评分 > 90

### 11. 日志检查 ⚠️ 关键

检查日志文件：
```bash
tail -n 50 /app/work/logs/bypass/app.log
tail -n 50 /app/work/logs/bypass/console.log
```

- [ ] **无 ERROR 级别日志**
- [ ] **无 Exception 或 Traceback**
- [ ] 无严重警告
- [ ] 日志格式正确
- [ ] 日志级别配置正确

### 12. 安全检查

#### 12.1 WAF 防护
- [ ] WAF 规则库已加载
- [ ] SQL 注入检测已启用
- [ ] XSS 攻击检测已启用
- [ ] 路径遍历检测已启用
- [ ] 命令注入检测已启用

#### 12.2 速率限制
- [ ] 速率限制已启用
- [ ] 白名单配置正确
- [ ] 黑名单配置正确
- [ ] 限制阈值合理

#### 12.3 日志审计
- [ ] 审计日志已启用
- [ ] 异常访问监控已启用
- [ ] 可疑 IP 检测正常
- [ ] 攻击尝试统计正常

#### 12.4 签名验证（可选）
- [ ] 核心接口签名验证已启用
- [ ] 密钥配置正确
- [ ] 时间戳验证正常
- [ ] Nonce 防重放正常

### 13. 监控检查

- [ ] 性能监控已启用（Web Vitals）
- [ ] 错误监控已启用（Sentry 或类似服务）
- [ ] 访问统计已启用（Analytics）
- [ ] 告警规则已配置
- [ ] 通知渠道已配置（邮件、短信等）

### 14. 备份检查

- [ ] 数据库备份已完成
- [ ] 代码备份已完成（Git 标签）
- [ ] 配置文件备份已完成
- [ ] 备份文件可访问
- [ ] 备份恢复测试通过

## 🚨 紧急回滚流程

如果部署后发现严重问题，执行以下回滚流程：

1. **停止服务**
   ```bash
   pkill -f "node.*next"
   ```

2. **恢复备份**
   ```bash
   # 恢复代码
   git checkout <previous-tag>

   # 恢复数据库
   pg_restore -d database_name backup_file.dump
   ```

3. **重新部署**
   ```bash
   pnpm install
   pnpm build
   pnpm start
   ```

4. **验证**
   - 检查服务状态
   - 检查日志
   - 检查功能

5. **通知团队**
   - 发送回滚通知
   - 记录回滚原因
   - 安排问题修复
